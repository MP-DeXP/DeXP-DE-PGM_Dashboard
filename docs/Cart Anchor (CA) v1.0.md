# Cart Anchor (CA) v1.0

Cart Anchor (CA) v1.0
― 구매 순간의 구조적 힘을 정의하는 프레임워크 ―
Abstract
기존의 커머스 분석은 “어떤 상품이 유입을 만들었는가” 또는
**“어떤 상품이 재구매를 만들었는가”**에 집중해 왔다.
그러나 실제 매출의 상당 부분은 이 두 질문 사이, 즉 구매 순간에 장바구니가 어떻게 구성되는가에서 발생한다.
본 문서는 Anchor Product Framework(APF) v1.0의 하위 확장으로서
Cart Anchor(CA) 개념을 정의하고,
상품이 동일 주문(cart) 내에서 다른 상품을 동반시키는 구조적 역할을
일관된 언어와 지표로 설명하기 위한 프레임워크를 제시한다.
CA는 유입(AA)이나 재구매(PCA)를 대체하지 않는다.
대신 APF를 Before–During–After의 입체적 구조로 완성시키는 ‘During’ 축을 담당한다.

1. 문제의식: “구매 순간”은 분석에서 비어 있었다
APF v1.0은 다음 두 질문에 답해왔다.
	•	AA (Acquisition Anchor)   → 이 상품은 고객을 브랜드로 데려오는가? 
	•	PCA (Purchase Chain Anchor)   → 이 상품은 이후 구매 사슬을 여는가? 
그러나 실무 현장에서는 다음 질문이 지속적으로 발생한다.
“이 상품을 장바구니에 넣는 순간,
고객은 왜 다른 상품도 함께 담는가?”
이 질문은 시간적 전이(AA → PCA)가 아니라,
동일 주문 내 동시 결합 구조에 관한 질문이다.
Cart Anchor는 바로 이 지점을 명시적으로 다루기 위해 도입되었다.

2. Cart Anchor(CA)의 정의
**Cart Anchor(CA)**란 다음과 같이 정의된다.
특정 상품이 포함된 주문에서,
동일 주문 내 다른 상품들의 동시 구매를
구조적으로 동반시키는 역할
여기서 중요한 점은 다음 두 가지다.
	•	구매 ‘이후’가 아니라, 구매 ‘순간(During)’을 다룬다. 
	•	인과를 주장하지 않는다. 
	•	“이 상품 때문에 다른 상품을 샀다”가 아니라 
	•	“이 상품이 포함될 때 장바구니 구조가 달라진다”는 관측된 결과를 다룬다. 

3. APF 내에서의 위치
CA는 기존 APF와 다음과 같이 정렬된다.
축
역할
시간적 위치
AA
유입의 시작점
Before
CA
장바구니 결합력
During
PCA
구매 사슬 형성
After
이를 하나의 문장으로 요약하면 다음과 같다.
AA는 입구의 넓이,
CA는 장바구니의 넓이,
PCA는 사슬의 길이다.
CA는 AA/PCA와 다른 차원의 역할이므로,
점수를 통합하거나 합산하지 않는다.

4. 분석 전제와 해석 경계
4.1 분석 단위
	•	주문 단위: order_id 
	•	상품 단위 집계: product_id 
4.2 데이터 전제
	•	취소 / 반품 / 교환 / 환불 제거된 정상 주문 기준 
	•	동일 주문 내 동일 상품은 1회로 계산 
	•	평균(avg)보다 median / percentile / rate 중심 사용   (heavy user 편향 방지) 
4.3 CA가 설명하지 않는 것
	•	고객의 재구매/잔존 → PCA 영역 
	•	신규 유입 → AA 영역 
	•	가격, 노출, 마케팅 개입 효과 → v1.1 이후 검증 대상 

5. CA Feature (구조 지표)
CA는 점수 이전에, 다음과 같은 **구조 지표(feature)**로 설명된다.
	•	Attach Rate   : 해당 상품이 포함된 주문 중, 다른 상품이 1개 이상 함께 담긴 비율 
	•	Median Cart Size   : 해당 상품이 포함된 주문의 상품 수 중앙값 
	•	Breadth Lift   : 해당 상품 주문의 median cart size − 전체 주문 median 
	•	Companion Count   : 해당 상품과 함께 등장한 서로 다른 product_id 수 
	•	Top1 / Top3 Share   : 동반 상품 중 상위 1개 / 3개가 차지하는 비중   (페어링/세트 집중도) 
CA feature는 결과 지표이며,
인과를 증명하기 위한 지표가 아니다.

6. CA 타입 분류 (룰 기반, 해석용)
CA는 점수화 이전에 해석과 기획을 위한 역할 타입으로 분류된다.
6.1 CA-Core (Cart Hub)
	•	다양한 상품을 함께 끌어오는 허브형 상품 
	•	attach_rate, breadth_lift, companion_cnt가 모두 높음 
	•	특정 1개 짝꿍에 과도하게 의존하지 않음 
의미:
번들 앞단, 추천 시작점, 장바구니 확장 레버

6.2 CA-Pair (Strong Companion)
	•	특정 1~2개 상품과의 결합이 압도적으로 강함 
	•	top1_share가 매우 높음 
	•	동일 상품 중복 구매가 아닌, 서로 다른 product_id 간 페어링 
의미:
고정 페어 번들, 강제 동반 추천에 최적

6.3 CA-Set (Bundle / Set Driver)
	•	다품목이 함께 담기는 세트형 구조 
	•	top3_share가 높고, cart size가 크게 증가 
의미:
이미 기획된 번들을 찾는 것이 아니라,
자생적으로 세트화된 구매 패턴을 발견하는 도구

6.4 Primary CA Type
여러 조건을 동시에 만족할 수 있으므로,
기획 주도권 기준으로 다음 우선순위를 둔다.
Pair → Set → Core → None

7. CA 기반 Product Transition 개념
CA의 전이는 시간적 전이가 아니라
동일 주문 내 동시 전이다.
	•	Node: product 
	•	Edge: 같은 주문에서 함께 등장 
	•	Weight: 
	•	P(j | i) (조건부 동시 구매 확률) 
	•	Lift (선택) 
이는 Cart Graph로 표현되며,
CA는 이 그래프에서 결합의 접점(node) 역할을 한다.

8. CA → PCA 연결 기획
CA는 단독 목적이 아니라,
PCA(사슬 개시 상품)로의 연결을 설계하기 위한 레버다.
예시:
	•	CA-Pair의 대표 companion이 PCA-Core일 경우   → AA → CA → PCA 경로를 의도적으로 설계 가능 
	•	CA-Core를 통해 다양한 상품을 노출한 뒤   → PCA-Core로 사슬을 열도록 유도 
이로써 APF는 다음의 자연스러운 경로를 갖는다.
AA (유입) → CA (장바구니 확장) → PCA (사슬 형성)

9. 해석 가이드 (필수 문구)
CA를 사용할 때 다음 문장을 반드시 전제로 한다.
“CA는 인과를 증명하지 않는다.
CA는 해당 상품이 포함될 때 장바구니 구조가 달라지는 현상을 포착한다.”
또한,
“높은 CA 점수는 재구매가 아니라,
장바구니 설계에 유리한 구조적 위치를 의미한다.”

10. Product Role 요약
최종적으로 상품은 다음 3차원 벡터로 해석된다.
Product Role = (AA, CA, PCA)
	•	AA: 유입력 
	•	CA: 장바구니 결합력 
	•	PCA: 사슬 형성력 
이 세 역할은 우열 관계가 아니라, 설계 레버의 차이다.

결론
Cart Anchor(CA)는 APF v1.0을 시간 중심 프레임워크에서 공간 중심 프레임워크로 확장한다.
CA를 통해 커머스 팀은 다음 질문에 명확히 답할 수 있다.
	•	어떤 상품이 장바구니를 키우는가? 
	•	어떤 상품이 번들의 출발점이 되는가? 
	•	어떤 조합이 자생적으로 형성되고 있는가? 
CA는 재구매를 예측하지 않는다.
CA는 구매 순간의 구조를 설명한다.
이 차이를 명확히 인식하는 한,
Cart Anchor는 APF의 가장 실무적인 확장 축이 될 것이다.
